# 观心开发计划

> 版本：v1.2
> 最后更新：2026-06-24

---

## 1. 计划原则

当前开发计划遵循四个原则：

- 先保证 `可用`
- 再保证 `可理解`
- 再保证 `可复访`
- 最后再扩展 `内容深度` 与 `社交可能性`

因此，未来一段时间不建议继续横向堆功能，而应优先：

- 收口现有体验
- 修复可用性问题
- 修复可理解性问题
- 验证第一阶段核心闭环
- 再启动真正的工程化拆分

## 2. 当前阶段判断

当前项目阶段：

**MVP 主体验已完成，第一阶段功能正在收口。**

当前最关键的问题不是“缺多少新功能”，而是：

- AI 是否稳定可用
- 用户是否能顺手返回上一页
- 零基础用户是否看得懂、知道如何开始
- AI 是否足够短、直、可执行
- 结果页是否明确告诉用户下一步
- 历史/收藏/7 天回看是否真的构成闭环
- 文档、实现和仓库结构是否一致
- 单文件架构是否开始进入可维护状态
- 新确认的“四维生活解析”和“术数表达三层法”是否真正落到页面

同时新增一条总约束：

- 术数架构已确认：`八卦为体，梅花为用，命宫 / 生日本命卦为人，五行为翻译层，节气为留存层`
- 八星八宅保留，但只做副层观察模块，不再继续扩大为主底层

## 3. 总体阶段规划

### 阶段 A：稳定可用与可理解

目标：

**先让当前版本对用户来说真正可用，而且第一次就能看懂。**

核心任务：

- 收口 AI 解读可用性
- 统一返回按钮位置与返回逻辑
- 为关键页面补符合直觉的手势返回
- 把新手引导改成零基础白话引导
- 将 AI 解读改为“短结论 + 行动建议”优先
- 给结果页补明确下一步引导
- 验证收藏是否真实可感
- 验证历史记录是否稳定
- 验证 7 天回看是否能完整触发与标记
- 对齐 README / PRD / 状态文档与真实实现

完成标准：

- 用户可以稳定完成一次完整问卦与 AI 解读
- 关键页面返回入口统一且顺手
- 零基础用户首次使用不需要额外解释也能完成主流程
- 用户看完结果后知道接下来该做什么
- 收藏、历史、回看三者可连贯使用
- 文档不再夸大当前完成度

### 阶段 B：第一阶段闭环完成

目标：

**让“情绪 → 启发 → 行动 → 回看”真正成立。**

核心任务：

- 优化 AI 行动卡表达和展示层级
- 落地四维生活解析（事业 / 感情 / 身心 / 钱财）
- 将命宫、本命卦、流年、八宅内容改写为术数表达三层法
- 让历史主题筛选真正服务用户理解自己
- 增强 7 天回看内容质量
- 提升首页回访理由的可见性
- 让结果页行动区与回看机制联动

完成标准：

- 用户不是“看完就走”，而是知道下一步怎么做
- 四维解析既保留术数依据，也能让零基础用户看懂
- 用户历史页中能看见自己的主要困扰主题
- 回看机制能帮助用户感知自己变化

### 阶段 C：工程化整理

目标：

**为后续长期迭代减轻技术债。**

核心任务：

- 拆分 `index.html`
- 提取卦象与命理数据
- 梳理目录结构
- 设计最小测试方案
- 减少未来修改单点风险

完成标准：

- 主应用不再完全依赖超大单文件
- 数据与视图初步分离
- 至少核心纯函数可做回归验证

### 阶段 D：内容深度建设

目标：

**让观心从功能型产品升级为内容型情绪成长产品。**

核心任务：

- 今日一问
- 多层解读模式
- 节气专题
- 月度回顾
- 人生场景图鉴增强

完成标准：

- 用户会因为内容本身回来，而不只是因为问卦
- 产品开始形成稳定的内容气质

### 阶段 E：轻社交验证

目标：

**在不破坏私密感和克制感的前提下，验证分享和共鸣是否成立。**

核心任务：

- 分享卡片 2.0
- 好友互占增强
- 一句回应
- 匿名主题观察

完成标准：

- 分享有真实传播场景
- 互动保持轻量，不把产品推向高噪声社区

## 4. 近期开发优先级

### P0：立刻推进

#### 1. AI 可用性稳定

为什么先做：

- 它是结果页最强感知能力之一
- 一旦不可用，用户会直接认为产品“坏了”

具体事项：

- 保证本地 Key 路径可用
- 保证代理兜底路径可用
- 优化错误提示文案
- 明确设置页使用说明

验收标准：

- 有 Key 时可稳定返回内容
- 无 Key 时提示清晰
- 错误时不出现技术性强、用户看不懂的提示

#### 2. 主路径导航与返回体验

为什么先做：

- 返回是所有页面的基础安全感
- 情绪类产品里，迷路和不敢点返回会直接降低信任

具体事项：

- 统一重要页面返回按钮到左上角
- 优先返回上一页，而不是默认返回首页
- 为关键页面补右滑返回上一页
- 明确页面层级，避免用户不知道自己在哪

验收标准：

- 用户单手也能快速找到返回
- 关键页面返回行为一致
- 滑动返回不会误触主交互

#### 3. 零基础新手引导重做

为什么先做：

- 当前最大流失风险不是功能缺失，而是看不懂和不会用

具体事项：

- 把功能导览改成“你现在适合点哪里”
- 给命宫、卦、爻、图鉴等术语补白话说明
- 在核心入口增加场景化提示
- 把“新手第一次该怎么用”做成最短路径

验收标准：

- 不懂周易的用户也能独立完成首次问卦
- 新手引导不依赖专业术语就能讲清主流程
- 用户知道不同入口分别适合什么问题

#### 4. AI 输出收敛与下一步引导

为什么先做：

- AI 现在的主要问题不是“有没有内容”，而是“没有重点、没有动作”

具体事项：

- AI 默认输出缩短为结论优先
- 优先呈现一句判断、一条提醒、一个行动
- 补“今天可以做什么 / 先不要做什么 / 何时回看”
- 压缩大段铺陈和重复解释

验收标准：

- 用户 5-10 秒内能抓到核心意思
- 每次解读都有明确下一步
- AI 文案整体更短、更直白、更不说教

## 5. 下一步功能计划

当前判断已经明确：

- 文本内容先不继续大扩写
- 下一步重点转到 `体验闭环`、`交互效率`、`回访理由`、`工程准备`

因此，接下来按 `P0 / P1 / P2` 三层推进。

### 5.1 P0：当前必须优先收口

目标：

**先把主路径做顺，让首页、命宫、问卦三条高频路径在手机端真正可用、可懂、可回访。**

具体要做：

- 命宫档案与八星磁场交互收口
  - 继续优化命宫档案页的信息密度。
  - 减少不必要的首屏文本占位。
  - 统一“点开查看 / 学习 / 深读”入口样式。
  - 继续修正八星磁场页点击反馈、选中态、展开态的辨识。
  - 检查弹层关闭按钮、滚动区、移动端可读性。
- 每日一卦回访闭环
  - 强化首页“今日状态板”和“今日练习”的存在感。
  - 把 `已记下 -> 待回看 -> 已完成` 做成更清楚的日常节奏。
  - 增加完成后的轻反馈文案或状态变化。
  - 检查隔天内容切换是否自然。
  - 让“回看昨天 / 看今天”之间关系更明确。
- 问卦结果页收口
  - 继续压缩结果页首屏。
  - 保证“先看结论 / 再看解释 / 再去学习”层级清楚。
  - 优化“先做什么 / 今天别做什么 / 什么时候回看”三个动作区。
  - 清理重复提示和重复卦义。
  - 继续校正分享卡片信息层级。
- 节气卡规则校准
  - 统一当前口径为“以日支为参考，再结合节气看当天节奏”。
  - 校正首页节气卡、弹层说明、标签文案与后续建议逻辑。

完成标准：

- 用户不需要猜哪里能点。
- 首页首屏像入口，不像说明板。
- 命宫页、结果页、八星页在手机端都能顺着走完。
- 用户能理解今天为什么回来。
- 节气卡规则前后一致，不再出现口径摇摆。

### 5.2 P1：形成回访和积累闭环

目标：

**让用户不是用完就走，而是愿意继续回看自己、积累轨迹。**

具体要做：

- 历史、收藏、主题档案形成闭环
  - 校验收藏反馈是否在所有入口都一致。
  - 强化历史页的主题筛选与回看价值。
  - 检查“同一天多次占卦”在历史中的阅读体验。
  - 设计下一步的“主题聚合页”雏形。
  - 评估是否要补一个“我最近常问什么”轻摘要。
- 节气留存层增强
  - 在现有最小版本基础上继续验证节气卡是否真能成为回访理由。
  - 明确节气页与每日一卦、首页之间的分工。
  - 节气内容保持轻量，不扩成复杂历法工具。
- 四维生活解析整理
  - 继续把事业 / 感情 / 身心 / 钱财四维入口收成统一结构。
  - 强化每一维的现实落点与边界提示。

完成标准：

- 用户能从历史里看到自己，不只是看到记录列表。
- 收藏、历史、主题三者关系更清楚。
- 节气不和每日一卦重复，而是承担另一种“顺时而看”的角色。

### 5.3 P2：工程化准备与后续扩展

目标：

**在不破坏当前单文件可运行状态的前提下，为后续拆分和商业化做准备。**

具体要做：

- 工程化准备
  - 继续校正文档和真实实现。
  - 梳理 `index.html` 内数据区、样式区、逻辑区的边界。
  - 提炼下一步最适合先拆的模块清单。
  - 先建立最小回归检查清单。
- 中长期内容层
  - 评估“今日一问”“月度回看”“多层解读模式”等中期内容能力。
  - 保持“八卦为体，梅花为用，命宫/本命卦为人，五行为翻译层，节气为留存层”的主链路不跑偏。

完成标准：

- 下一轮如果进入拆分，不会边拆边乱。
- 现阶段单文件仍可继续稳定迭代。
- 后续新功能不会再把主路径拉散。

验收标准：

- 每段解读至少保留一个明确术数依据。
- 每段解读至少给一个现实核对点或行动点。

### 5.1 当前执行批次：阶段 A / P0 收口

本批次目标不是扩展新功能，而是修复朋友测试暴露出的基础体验问题，让用户第一次使用时能顺利完成主流程。

执行切片：

1. 返回体验：统一关键页面左上返回，内部导航优先返回上一页，并在移动端关键页面支持右滑返回。

### 5.2 二次修正后的验收顺序

最新反馈说明：只做“有返回、有白话、有行动卡”还不够，必须让用户明显感觉到页面在帮他理解和行动。

### 5.3 第三次修正后的验收顺序

5. 图鉴入口：结果页负责“当下怎么理解和行动”，图鉴负责“传统结构和延展学习”，二者内容不能重复。
