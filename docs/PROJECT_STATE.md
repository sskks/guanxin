## 1. 当前判断

项目已经完成了 `可体验 MVP` 的主体框架，但还不能简单定义为“全部完成并稳定上线”。

从产品方向上看，`观心` 当前应明确定位为：

- 以情绪陪伴为核心的产品
- 以优秀传统文化为内容底座的启发式产品
- 帮助用户把“领悟”转化为现实生活实践的自我成长产品

更准确地说，当前状态是：

- 核心单人体验已经打通
- 第一阶段留存能力已经部分落地
- 文档体系已经建立
- 项目级 `PRODUCT.md` / `DESIGN.md` 已建立
- 前后端商业化演进骨架已建立
- AI 可用性、首次可理解性、导航一致性仍然是明显短板
- 测试覆盖和工程拆分仍然是明显短板

因此，当前更适合定义为：

**MVP 已可运行，正在进行第一阶段收口与内容表达升级。**

### 3.7 占卦历史覆盖专项修复（2026-06-22）

本轮针对“占卦历史会覆盖”的反馈做了专项收口：

- `IndexedDB 记录模型`
  - 新增精确主键读取函数，收藏、七日回看、AI 结果回写不再通过日期索引兜底查找。
  - 新记录继续使用 `mei_...` / `num_...` 唯一 key 作为 `date` 主键，`day` 只作为日期分组字段。
  - 旧记录仍保留兼容读取，不强制迁移，避免破坏用户已有历史。

- `AI 解读回写`
  - AI 解读只在存在当前记录唯一 key 时写回。
  - 好友分享、无本地记录或旧自然日期参数不会再回退到“当天日期”覆盖历史。

- `收藏与历史`
  - 历史列表的收藏操作只按当前卡片的唯一记录 key 更新，不再按 `day` 索引取同日第一条。
  - 已验证同一天连续两次数字起卦会进入历史页并同时显示。

已验证：

- `index.html` 内联脚本语法检查通过。
- 本地 `http://127.0.0.1:8080/` 返回 HTTP 200。
- 浏览器实际页面验证：历史页渲染出两条同日测试记录，记录问题文本分别保留。

### 3.25 四维解析敏感表达边界确认（2026-06-23）

基于“四维解析会触及事业、感情、身心、钱财等敏感主题，不能制造恐慌，也不能掩盖不说”的反馈，本轮进一步收口内容安全标准：

- `总原则`
  - 四维解析必须客观但不冷漠：可以指出卡点和风险，但不能用“注定、必然、大凶、必破、必分”等断言。
  - 不能恐吓，也不能粉饰。表达应落在“这里需要留意什么、现实里先核对什么、今天能先做什么”。
  - 术数只作为观察镜子，不替代医疗、财务、法律、亲密关系等现实判断。
- `维度边界`
  - 事业：讲行动节奏、压力、协作和边界；不预测升职、裁员、考试、创业成败。
  - 感情：讲表达、边界、期待落差和修复；不判断对方爱不爱、是否出轨、是否复合、是否正缘。
  - 身心：讲压力负荷、睡眠节律、情绪消耗和自我照料；不诊断疾病、不解释病因、不替代医生或心理咨询。
  - 钱财：讲消费节奏、资源管理、安全感和风险意识；不预测发财破财，不推荐金融产品，不做投资建议。
- `固定输出结构`
  - 每一维必须包含：一句客观观察、可能卡点、现实核对、今日练习、传统根据、边界提醒。
  - 身心维度如果涉及持续痛苦、睡眠饮食明显受影响、无法完成日常任务等情况，应提示寻求专业帮助。

### 3.26 术数表达三层法确认（2026-06-23）

基于“文字输出需要在温和修饰与术数本意之间取得平衡”的反馈，本轮确认后续内容表达方法：

- `核心方法`
  - 采用“术数骨架 -> 现实翻译 -> 情绪承接”的三层表达。
  - 先保留卦象、爻位、五行、八星、命宫等传统依据，再翻译成现实处境，最后给出可行动的承接。
- `避免两种偏差`
  - 不能过度修饰成普通心理鸡汤，否则会失去传统文化术数的魅力和准确感。
  - 不能直接使用恐吓式断语，否则会放大焦虑，削弱产品的情绪安顿价值。
- `验收标准`
  - 每段解读至少保留一个明确传统依据。
  - 每段解读必须指出一个真实卡点或现实核对点。
  - 每段解读必须给一个低风险、当天可做的小行动。
  - 敏感主题必须保留边界提醒。

## 5. 已知问题

### 高优先级

- AI 能力依赖用户本地 Key 或代理配置，稳定性仍不足
- 返回入口不统一，部分页面更像“回首页”而不是“回上一页”
- 新手理解门槛偏高，术语解释与场景引导不足
- AI 结果虽然已有结构化卡片，但仍存在冗长、不够动作导向的问题
- 结果页缺少明确下一步建议，闭环仍不够强
- 当前运行入口仍是超大单文件 `index.html`，虽然仓库已有 `css/`、`js/`、`data/` 目录，但拆分尚未真正落地
- 尚无自动化测试，核心算法正确性无法持续回归验证

### 中优先级

- 历史、收藏、7 天回看缺少系统化验收
- 无障碍支持不足
- 页面性能仍有优化空间

### 低优先级

- 补丁脚本仍保留在仓库中，需后续整理归档
- 文档部分内容仍需要持续校正
