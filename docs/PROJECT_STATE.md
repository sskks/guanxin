## 1. 当前判断

项目已经完成了 `可体验 MVP` 的主体框架，但还不能简单定义为“全部完成并稳定上线”。

从产品方向上看，`观心` 当前应明确定位为：

- 以情绪陪伴为核心的产品
- 以优秀传统文化为内容底座的启发式产品
- 帮助用户把“领悟”转化为现实生活实践的自我成长产品

更准确地说，当前状态是：

- 核心单人体验已经打通
- 第一阶段留存能力已经部分落地
- 文档体系已经建立
- 项目级 `PRODUCT.md` / `DESIGN.md` 已建立
- 前后端商业化演进骨架已建立
- AI 可用性、首次可理解性、导航一致性仍然是明显短板
- 测试覆盖和工程拆分仍然是明显短板

因此，当前更适合定义为：

**MVP 已可运行，正在进行第一阶段收口与稳定化。**

### 3.3 最新用户测试反馈（2026-06-22）

朋友测试反馈已明确暴露出以下问题：

- 返回按钮位置不统一，且缺少顺手的返回上一页体验

- 是否好返回

### 3.4 本轮已开始实现（2026-06-22）

围绕用户反馈，当前已在 `index.html` 完成第一批 P0 收口改动：

- 新增内部导航栈，返回优先回到上一页，而不是默认回首页。
- 在占卦结果、每日一卦、摇签、历史、命宫解读、卦象图鉴等关键页面接入右滑返回上一页。

- 命宫解读动态页面中的返回按钮已从“返回首页”调整为“返回上一页”，无历史时兜底回首页。
- 新手引导从功能说明改为零基础白话引导，增加“卦、爻、命宫”的大白话解释。
- AI prompt 已从“大师长文解读”调整为“情绪产品里的传统文化解读助手”，要求短句、直接、可执行。

- 结果页“先做这一件事”已支持根据默认总结或 AI 行动建议动态更新。

待验收：

- 需要用真实 Qwen Key 或后端代理验证 AI 返回格式、长度和行动卡展示。

### 3.5 二次体验修正（2026-06-22）

基于最新测试反馈，第一批 P0 收口仍存在“变化不明显、解释不够深、分享弱、跳转不精准”的问题。本轮已继续修改：

- `命理档案页`
  - 新增左上角返回栏，不再只依赖页面底部入口。
  - 新增“先用白话看懂自己”卡片，解释命宫、本命卦、流年宫、五行生克分别和用户有什么关系。
  - 增加现实建议，不再只展示术语和长段传统描述。

- `占卦结果页`
  - 保留左上角返回，移除底部“返回上一页”按钮，避免返回入口出现在页面最底部。
  - 新增“先用白话看懂”卡片，用普通话解释当前卦不是判决，而是帮助用户分辨处境。
  - 新增动爻、互卦、变卦的大白话解释。
  - “先做这一件事”改为可执行动作，不再只是压缩卦辞或象辞。

- `卦象图鉴`
  - 结果页“看懂这个卦”改为直接打开当前卦详情，不再跳到图鉴列表让用户自己找。
  - 图鉴详情新增“白话先看”和“术语怎么理解”，降低零基础用户理解门槛。

- `分享能力`
  - 微信/系统分享文案从一行字改为结构化内容：卦名、问题、白话启发、今天先做的一件事、互卦/变卦延伸观察。
  - 分享卡片 UI 重做，突出“白话启发”和“今天先做这一件事”，弱化旧式卦辞堆叠。

- `AI 解读`
  - prompt 进一步要求“建议行动”必须是今天能完成的小动作，禁止只复述卦辞或使用“顺势、谨慎、等待”等空泛词。

### 3.7 占卦历史覆盖专项修复（2026-06-22）

本轮针对“占卦历史会覆盖”的反馈做了专项收口：

- `IndexedDB 记录模型`
  - 新增精确主键读取函数，收藏、七日回看、AI 结果回写不再通过日期索引兜底查找。
  - 新记录继续使用 `mei_...` / `num_...` 唯一 key 作为 `date` 主键，`day` 只作为日期分组字段。
  - 旧记录仍保留兼容读取，不强制迁移，避免破坏用户已有历史。

- `AI 解读回写`
  - AI 解读只在存在当前记录唯一 key 时写回。
  - 好友分享、无本地记录或旧自然日期参数不会再回退到“当天日期”覆盖历史。

- `收藏与历史`
  - 历史列表的收藏操作只按当前卡片的唯一记录 key 更新，不再按 `day` 索引取同日第一条。
  - 已验证同一天连续两次数字起卦会进入历史页并同时显示。

已验证：

- `index.html` 内联脚本语法检查通过。
- 本地 `http://127.0.0.1:8080/` 返回 HTTP 200。
- 浏览器实际页面验证：历史页渲染出两条同日测试记录，记录问题文本分别保留。

遗留说明：

- 当前产品还没有“删除单条历史”功能，本轮浏览器验证产生的本地测试记录无法通过 UI 清理；后续可在历史功能迭代中补删除/清空能力。

### 3.8 摇签翻转卡片位置修复（2026-06-22）

## 5. 已知问题

### 高优先级

- AI 能力依赖用户本地 Key 或代理配置，稳定性仍不足
- 返回入口不统一，部分页面更像“回首页”而不是“回上一页”
- 新手理解门槛偏高，术语解释与场景引导不足
- AI 结果虽然已有结构化卡片，但仍存在冗长、不够动作导向的问题
- 结果页缺少明确下一步建议，闭环仍不够强
- 当前运行入口仍是超大单文件 `index.html`，虽然仓库已有 `css/`、`js/`、`data/` 目录，但拆分尚未真正落地
- 尚无自动化测试，核心算法正确性无法持续回归验证

### 中优先级

- 历史、收藏、7 天回看缺少系统化验收
- 无障碍支持不足
- 页面性能仍有优化空间

### 低优先级

- 补丁脚本仍保留在仓库中，需后续整理归档
- 文档部分内容仍需要持续校正
