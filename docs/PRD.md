# 观心 PRD

> 版本：v1.2 MVP | 最后更新：2026-06-26

---

## 1. 产品定义

**一句话定义**：一个以优秀传统文化为内容底座、以情绪陪伴为核心体验、帮助用户拨开迷雾、安顿情绪并把领悟落实到现实生活中的东方哲思产品。

`观心` 不是传统意义上的算命工具，也不是强调“准不准”的玄学产品，而是一个结合优秀传统文化、轻仪式感、启发式内容和 AI 解读的情绪陪伴与自我成长产品。它的核心价值不是替用户做决定，而是帮助用户在焦虑和混乱中整理情绪、在关系和选择中获得启发、在长期使用中形成反思习惯，并将领悟到的道理和学到的知识应用到现实生活中，以更稳健的心态、更清晰的三观去面对生活。

---

## 2. 为什么做

### 2.1 用户需求

当前用户在“情绪整理”和“自我反思”方面有稳定需求，但市场上常见产品存在两个问题：传统命理产品过重、过玄、学习门槛高；现代情绪陪伴产品缺乏东方文化审美与哲思深度。用户真正需要的不是“一个标准答案”，而是一个温和的切入口、一种有仪式感的思考方式、一套可回看可持续的自我观察机制。

### 2.2 市场机会

观心适合切入东方文化内容产品、轻情绪陪伴产品、自我成长与 journaling 产品、AI 驱动的启发式内容工具等交叉赛道。增长点不在于功能堆叠，而在于是否能形成复访习惯、是否有独特的内容气质、是否能在“启发”与“边界感”之间取得平衡。

---

## 3. 目标用户

### 3.1 核心人群

20-35 岁，对情绪、自我成长、关系议题敏感的用户。喜欢东方文化，但不想接触过重术数体系。遇到选择、关系、工作困惑时，希望获得启发式建议。

### 3.2 用户特征

- 愿意接受轻仪式感和情绪化表达
- 需要内容可读、可感、不过度专业
- 不一定相信“预测”，但愿意接受“反思工具”
- 对产品气质、视觉审美和语言风格敏感
- 很多用户对周易、术语、卦象没有前置知识

---

## 4. 核心使用场景

**情绪整理**：“我最近很乱，但说不上来问题在哪。” 用户希望通过问卦、每日一卦或命宫内容，快速进入一种更清晰的内在状态。

**决策参考**：“我正面临一个选择，想获得不是鸡汤、也不是命令式的建议。” 用户需要的是启发式思考框架，而不是替代判断。

**长期反思**：“我想知道这段时间我一直在反复困扰什么。” 用户需要通过历史记录、主题归档和回看机制认识自己。

**日常学习与修身**：“我愿意每天花一点时间，获得一个有东方意味的提醒。” 由每日一卦、节气内容、今日一问等能力承接。

---

## 5. 产品原则

**启发而不替代**：产品应提供理解与提醒，不替用户做现实决策。

**有哲思，但不故作高深**：语言可以有文化感，但不能让普通用户读不懂。

**有温度，但有边界**：情绪陪伴要克制，不制造过度依赖感，也不强化宿命论。

**仪式感服务于复访**：视觉、交互和内容都要强化“回来再看”的价值。

**先白话，再术语**：先让用户知道这是什么意思、和自己有什么关系，再展示完整周易术语与出处。

**先行动，再长文**：结果输出必须优先告诉用户重点与下一步，不默认给大段解释。

**先保留术数骨架，再做情绪承接**：卦象、爻位、五行、八星、命宫等传统依据不能被全部稀释成普通安慰话术；但传统断语也必须转译成倾向、处境和行动提醒，避免恐吓和宿命论。

**术数主链路固定**：八卦为体，梅花为用，命宫 / 生日本命卦为人，五行为翻译层，节气为留存层。所有新增功能优先服务这条主链路，不再并列扩展多个重术数系统。

**先可返回，再可探索**：在情绪场景中，用户必须随时能顺手回到上一步，导航安全感优先于功能丰富度。

## 6. 功能清单

### P0 - 核心功能

**每日一卦**：根据当前日期自动展示对应卦象，包含卦图、卦辞、象辞，支持前后日翻页。验收标准：打开页面自动显示当日卦象；左右滑动可切换日期；卦辞和象辞完整展示；零基础用户能看懂“今天在说什么”。

**竹签摇卦**：点击摇卦按钮或物理摇晃手机，播放竹签动画后随机生成卦象，以 3D 翻牌方式揭晓结果。验收标准：动画流畅无卡顿；支持 DeviceMotion 物理摇晃触发（移动端）；结果卦象信息完整。

**问易（以事问卦）**：用户输入心中所惑的问题，系统以梅花易数算法基于毫秒级时间戳生成卦象，并自动进入结果页。验收标准：输入问题后可生成卦象；结果包含卦图、卦辞、变爻；用户知道这一步是“借问题看自己当下状态”。

**数字起卦**：用户输入两个数字，上卦=数1%8、下卦=数2%8、变爻=(数1+数2)%6，生成卦象。验收标准：两个数字输入框可用；计算逻辑正确；结果展示完整；入口有白话说明，告诉用户适合什么场景。

**占卦结果页**：统一的结果展示页，包含左上角返回、卦图（CSS 绘制阴阳爻）、卦名、卦辞、象辞、变爻高亮、AI 解读卡片、下一步行动区、分享工具栏（复制文字/原生分享/保存图片）。验收标准：所有占卦方式共享此结果页；返回行为优先回到上一页；AI 解读以结构化“行动卡”格式渲染；页面明确展示“今天可做 / 先别做 / 何时回看”；分享功能三路径均可用。

**全局返回体验**：关键功能页统一使用左上角返回按钮，并在移动端关键路径支持右滑返回上一页。验收标准：结果页、图鉴页、历史页、八星磁场页等返回位置一致；返回逻辑优先回上一页而不是首页；手势返回不影响页面主交互。

### P1 - 重要功能

**命宫档案**：基于用户出生信息（年/月/日/时辰），生成命宫、流年宫与生日本命卦参考，在首页以卡片形式展示。验收标准：生日录入表单完整（年月日+十二时辰）；命宫、流年宫、生日本命卦计算逻辑可解释；命宫描述和流年分析展示完整；首屏有白话摘要，不让用户直接被术语淹没。当前版本不承诺完整四柱八字排盘。

**四维生活解析**：在命宫档案页新增事业、感情、身心、钱财四个维度，把命宫、生日本命卦、流年宫、八星磁场和五行关系转译为现代生活问题。验收标准：每一维包含一句白话判断、一个常见卡点、一个现实核对问题、一个今日练习、一个传统根据和一个边界提醒；身心维度不做医疗判断，钱财维度不做投资建议；感情维度不判断对方爱不爱、是否出轨或是否复合；事业维度不预测升职、裁员、考试、创业成败；页面明确这不是紫微斗数命盘，也不是八字十神排盘。

**八星磁场页**：基于命宫数与八方组合关系，以互动盘形式展示八星对应的性格倾向、关系磁场和近期状态提醒。验收标准：盘面方位正确；用户能看懂“命宫数 + 方位数 = 八星”的关系；页面明确说明这是借八星做自我观察，不提供阳宅风水判断；文案以“助力 / 提醒 / 失衡点 / 现实用法”为主，而不是直接做吉凶断语。

**图鉴百科**：以网格形式展示全部六十四卦，支持按上卦/下卦筛选，点击可查看卦辞、象辞和六条爻辞。验收标准：64 卦全部展示无遗漏；筛选功能正确过滤；详情弹窗信息完整；每卦都至少有白话版摘要。

**占卦历史**：列表展示所有过往占卦记录，支持主题筛选（事业/感情/成长/情绪/其他）、收藏标记、月度报告生成。验收标准：记录按时间倒序排列；收藏可切换；主题筛选正确过滤；用户能看出自己最近反复困扰的主题。

**七日回顾**：占卦满 7 天后，在首页展示回顾卡片，用户可查看原始问题和卦象，标记“已验证”。验收标准：7 天后正确弹出回顾记录；标记后不再重复展示；`reviewedAt` 字段正确写入 IndexedDB；回顾内容能帮助用户看到“当时说了什么、现在发生了什么”。

### P2 - 辅助功能

**每日签到**：首页签到按钮，连续签到天数以徽章展示。验收标准：每日限签到一次；连续天数计算正确；数据持久化。

**AI 解读**：将卦象数据 + 用户问题发送至通义千问（qwen-plus），返回结构化解读，默认采用“先结论、再行动、后解释”的顺序，包含整体判断、当前提醒、建议行动、今日避免、一句话提醒。支持服务端代理和用户自填 API Key 直连两种调用方式。验收标准：解读内容格式化为行动卡；首屏 5-10 秒内可抓到重点；文案整体更短更白话；错误时展示友好提示（覆盖未配置 Key、代理不可达、401、404 等场景）；两种调用路径均可用。

**好友分享**：生成包含占卦数据的分享链接，接收方打开后可查看只读结果。验收标准：链接可正常打开；接收方看到完整卦象信息；无需登录即可查看。

**新手引导**：首次打开时展示零基础白话引导，优先解释“你适合从哪里开始”“这个功能适合什么问题”“看完后怎么用”。验收标准：仅首次展示；可跳过；不用专业术语也能讲清主路径；用户看完知道首个推荐入口。

**设置**：允许用户输入个人 DashScope API Key，用于浏览器直连 AI 解读。验收标准：Key 保存后下次调用生效；提供使用说明。

## 7. 核心用户流程

**流程一：首次使用** → 启动页 → 生日录入 → 加载动画 → 命宫档案首屏（完成第一次“看懂自己”）→ 返回首页继续主路径

**流程一补充原则**：首次建档后优先进入命宫档案页是为了完成“个人化第一印象”和新手白话承接；但这条路径只服务首次或重新建档，不应成为日常回访的默认入口。

**流程二：以事问卦** → 首页 → 点击“问易” → 输入问题 → 竹签摇卦动画 → 3D 翻牌揭晓 → 占卦结果页 → AI 解读 → 下一步行动提示 → 可选择收藏、回看或分享

**流程三：每日学习** → 首页 → 点击每日一卦 / 节气内容 → 查看今日短层 → 按需进入深读 → 记下今日练习 → 返回首页

**流程四：回顾历史** → 首页 → 点击历史 → 筛选 / 收藏 → 查看月度报告 → 点击记录进入结果页

## 8. 视觉风格

整体风格为中国古风水墨画：纸质纹理背景、墨色主调、书法字体标题、土色系配色。设计系统基于 CSS 自定义属性，响应式适配（768px / 480px 断点）。动画风格克制优雅，服务于安顿感，而不是炫技。

## 9. 路线图

### 第一阶段：单人体验闭环（当前）

目标：让第一次来的人愿意留下，让来过的人愿意再回来。

当前阶段要从“功能可用”升级为“看得懂、回得去、知道下一步”。已完成 AI 解读行动卡、7 天回看机制、主题档案、AI 错误提示优化；下一步优先统一返回体验、重做新手引导、收敛 AI 输出、补强结果页行动引导。

同时确认一个新的首页策略：`首次建档看命宫，日常回访先看每日变化`。首页不再只是功能分发页，而要承担“今天为什么值得回来”的职责，优先展示每日一卦、今日练习、七日回看、签到和后续节气内容。

### 第二阶段：内容深度与差异化

目标：让用户觉得这是一个“有思想”的产品。规划功能：今日一问、多层解读模式（白话 / 哲思 / 行动 / 清醒提醒）、四维生活解析（事业 / 感情 / 身心 / 钱财）、人生场景图鉴、节气专题、月度回顾。

### 第三阶段：轻社交验证

目标：验证分享和轻互动是否成立，但不急于做重社区。规划功能：分享卡片 2.0、好友互占升级、一句回应、匿名主题观察。

## 10. 指标设计

**第一阶段**：生辰信息完成率、首次问卦完成率、AI 解读点击率、3 日回访率、7 日回看触发率、历史记录打开率。

补充重点指标：结果页返回成功率、新手引导完成率、AI 解读首屏阅读完成率、结果页行动区点击率。

**第二阶段**：图鉴停留时长、收藏率、月度回顾打开率、每日一卦连续打卡率。

**第三阶段**：分享率、分享后打开率、好友互占使用率、匿名主题浏览率。

## 11. 非目标（当前阶段不做）

重社区、评论系统、关注系统、复杂术数堆叠、强“准不准”叙事、紫微斗数正式排盘、八字十神正式排盘。原因：容易削弱产品主价值、容易带来内容失控与情绪放大、不利于建立克制可信的品牌调性。紫微斗数和八字十神可作为后续研究方向，但当前版本只借鉴“按人生问题分区”的产品结构，不对外承诺专业命盘。

## 12. 风险与边界

**内容风险**：用户可能将结果当作现实决策依据，AI 解读可能出现过度主观表达。应对：前置免责声明、保持“启发式”语言、增加边界提醒。

**敏感维度风险**：事业、感情、身心、钱财容易触及用户焦虑和真实痛点。应对：四维解析必须使用“观察倾向 + 现实核对 + 低风险行动 + 边界提醒”的结构；不得恐吓，不得粉饰，不得做医疗、投资、法律或关系断言；出现自伤、自杀、严重身心困扰等信号时，应提示寻求专业帮助或当地紧急支持。

**产品风险**：功能过多导致主价值分散、内容太玄导致普通用户流失。应对：所有新功能优先服务主路径、每次版本只强化一个核心闭环。

**理解风险**：术语太多、引导太少，导致零基础用户直接流失。应对：白话优先、场景优先、术语后置。

**社交风险**：公开内容可能引发隐私与情绪安全问题。应对：先做轻分享再考虑轻互动、避免过早开放广场和评论。

---

## 13. 技术约束

纯前端单文件 SPA，无框架依赖，无构建工具，无后端服务（AI 解读代理除外）。数据持久化使用 IndexedDB（占卦记录）和 localStorage（用户设置）。部署于 Cloudflare Pages / GitHub Pages，支持纯静态托管。
