# 观心 PRD

> 版本：v1.0 MVP | 最后更新：2026-06-22

---

## 1. 产品定义

**一句话定义**：一个以周易语言帮助用户整理情绪、理解关系、思考选择的东方哲思陪伴产品。

`观心` 不是传统意义上的算命工具，也不是强调"准不准"的玄学产品，而是一个结合东方文化、轻仪式感、启发式内容和 AI 解读的自我观察产品。它的核心价值不是替用户做决定，而是帮助用户在焦虑和混乱中整理思路、在关系和选择中获得启发、在长期使用中形成反思习惯、在回看中理解自己的变化。

---

## 2. 为什么做

### 2.1 用户需求

当前用户在"情绪整理"和"自我反思"方面有稳定需求，但市场上常见产品存在两个问题：传统命理产品过重、过玄、学习门槛高；现代情绪陪伴产品缺乏东方文化审美与哲思深度。用户真正需要的不是"一个标准答案"，而是一个温和的切入口、一种有仪式感的思考方式、一套可回看可持续的自我观察机制。

### 2.2 市场机会

观心适合切入东方文化内容产品、轻情绪陪伴产品、自我成长与 journaling 产品、AI 驱动的启发式内容工具等交叉赛道。增长点不在于功能堆叠，而在于是否能形成复访习惯、是否有独特的内容气质、是否能在"启发"与"边界感"之间取得平衡。

---

## 3. 目标用户

### 3.1 核心人群

20-35 岁，对情绪、自我成长、关系议题敏感的用户。喜欢东方文化，但不想接触过重术数体系。遇到选择、关系、工作困惑时，希望获得启发式建议。

### 3.2 用户特征

愿意接受轻仪式感和情绪化表达；需要内容可读、可感、不过度专业；不一定相信"预测"，但愿意接受"反思工具"；对产品气质、视觉审美和语言风格敏感。

---

## 4. 核心使用场景

**情绪整理**："我最近很乱，但说不上来问题在哪。" 用户希望通过问卦、每日一卦或命宫内容，快速进入一种更清晰的内在状态。

**决策参考**："我正面临一个选择，想获得不是鸡汤、也不是命令式的建议。" 用户需要的是启发式思考框架，而不是替代判断。

**长期反思**："我想知道这段时间我一直在反复困扰什么。" 用户需要通过历史记录、主题归档和回看机制认识自己。

**日常学习与修身**："我愿意每天花一点时间，获得一个有东方意味的提醒。" 由每日一卦、节气内容、今日一问等能力承接。

---

## 5. 产品原则

**启发而不替代**：产品应提供理解与提醒，不替用户做现实决策。

**有哲思，但不故作高深**：语言可以有文化感，但不能让普通用户读不懂。

**有温度，但有边界**：情绪陪伴要克制，不制造过度依赖感，也不强化宿命论。

**仪式感服务于复访**：视觉、交互和内容都要强化"回来再看"的价值。

## 6. 功能清单

### P0 - 核心功能

**每日一卦**：根据当前日期自动展示对应卦象，包含卦图、卦辞、象辞，支持前后日翻页。验收标准：打开页面自动显示当日卦象；左右滑动可切换日期；卦辞和象辞完整展示。

**竹签摇卦**：点击摇卦按钮或物理摇晃手机，播放竹签动画后随机生成卦象，以 3D 翻牌方式揭晓结果。验收标准：动画流畅无卡顿；支持 DeviceMotion 物理摇晃触发（移动端）；结果卦象信息完整。

**问易（以事问卦）**：用户输入心中所惑的问题，系统以梅花易数算法基于毫秒级时间戳生成卦象，并可选 AI 解读。验收标准：输入问题后可生成卦象；结果包含卦图、卦辞、变爻；AI 解读格式化为"行动卡"样式。

**数字起卦**：用户输入两个数字，上卦=数1%8、下卦=数2%8、变爻=(数1+数2)%6，生成卦象。验收标准：两个数字输入框可用；计算逻辑正确；结果展示完整。

**占卦结果页**：统一的结果展示页，包含卦图（CSS 绘制阴阳爻）、卦名、卦辞、象辞、变爻高亮、AI 解读卡片、分享工具栏（复制文字/原生分享/保存图片）。验收标准：所有占卦方式共享此结果页；AI 解读以结构化"行动卡"格式渲染；分享功能三路径均可用。

### P1 - 重要功能

**命宫解读**：基于用户出生信息（年/月/日/时辰），计算八字、本命卦、命宫、流年，在首页以卡片形式展示。验收标准：生日录入表单完整（年月日+十二时辰）；八字计算逻辑正确；命宫描述和流年分析展示完整。

**八宅风水**：根据用户本命卦计算八宅布局，以罗盘式界面展示八方位对应的游星及五行关系。验收标准：罗盘布局方位正确；吉凶游星颜色区分清晰；五行生克关系解释完整。

**图鉴百科**：以网格形式展示全部六十四卦，支持按上卦/下卦筛选，点击可查看卦辞、象辞和六条爻辞。验收标准：64卦全部展示无遗漏；筛选功能正确过滤；详情弹窗信息完整。

**占卦历史**：列表展示所有过往占卦记录，支持主题筛选（事业/感情/成长/情绪/其他）、收藏标记、月度报告生成。验收标准：记录按时间倒序排列；收藏可切换；主题筛选正确过滤；月度报告包含统计数据。

**七日回顾**：占卦满7天后，在首页展示回顾卡片，用户可查看原始问题和卦象，标记"已验证"。验收标准：7天后正确弹出回顾记录；标记后不再重复展示；reviewedAt 字段正确写入 IndexedDB。

### P2 - 辅助功能

**每日签到**：首页签到按钮，连续签到天数以徽章展示。验收标准：每日限签到一次；连续天数计算正确；数据持久化。

**AI 解读**：将卦象数据 + 用户问题发送至通义千问（qwen-plus），返回结构化解读（整体寓意、当前形势、建议行动、今日避免、一句话提醒）。支持服务端代理和用户自填 API Key 直连两种调用方式。验收标准：解读内容格式化为行动卡；错误时展示友好提示（覆盖未配置 Key、代理不可达、401、404 等场景）；两种调用路径均可用。

**好友分享**：生成包含占卦数据的分享链接，接收方打开后可查看只读结果。验收标准：链接可正常打开；接收方看到完整卦象信息；无需登录即可查看。

**新手引导**：首次打开时展示多步骤教程覆盖层，介绍核心功能。验收标准：仅首次展示；可跳过；步骤指示器正确。

**设置**：允许用户输入个人 DashScope API Key，用于浏览器直连 AI 解读。验收标准：Key 保存后下次调用生效；提供使用说明。

## 7. 核心用户流程

**流程一：首次使用** → 启动页 → 生日录入 → 加载动画 → 新手引导 → 首页（展示八字摘要、本命卦、每日一卦、功能导航卡片）

**流程二：以事问卦** → 首页 → 点击"问易" → 输入问题 → 竹签摇卦动画 → 3D 翻牌揭晓 → 占卦结果页 → 可选 AI 解读 → 分享或返回首页

**流程三：每日学习** → 首页 → 点击每日一卦 → 查看卦象信息 → 签到打卡 → 返回首页

**流程四：回顾历史** → 首页 → 点击历史 → 筛选/收藏 → 查看月度报告 → 点击记录进入结果页

---

## 8. 视觉风格

整体风格为中国古风水墨画：纸质纹理背景、墨色主调、书法字体标题、土色系配色。设计系统基于 CSS 自定义属性，响应式适配（768px / 480px 断点）。动画风格克制优雅——水墨晕染、竹签轻摇、翻牌翻转。

---

## 9. 路线图

### 第一阶段：单人体验闭环（当前）

目标：让第一次来的人愿意留下，让来过的人愿意再回来。已完成 AI 解读行动卡、7 天回看机制、主题档案、AI 错误提示优化。

### 第二阶段：内容深度与差异化

目标：让用户觉得这是一个"有思想"的产品。规划功能：今日一问、多层解读模式（白话/哲思/行动/清醒提醒）、人生场景图鉴、节气专题、月度回顾。

### 第三阶段：轻社交验证

目标：验证分享和轻互动是否成立，但不急于做重社区。规划功能：分享卡片 2.0、好友互占升级、一句回应、匿名主题观察。

---

## 10. 指标设计

**第一阶段**：生辰信息完成率、首次问卦完成率、AI 解读点击率、3 日回访率、7 日回看触发率、历史记录打开率。

**第二阶段**：图鉴停留时长、收藏率、月度回顾打开率、每日一卦连续打卡率。

**第三阶段**：分享率、分享后打开率、好友互占使用率、匿名主题浏览率。

---

## 11. 非目标（当前阶段不做）

重社区、评论系统、关注系统、复杂术数堆叠、强"准不准"叙事。原因：容易削弱产品主价值、容易带来内容失控与情绪放大、不利于建立克制可信的品牌调性。

---

## 12. 风险与边界

**内容风险**：用户可能将结果当作现实决策依据，AI 解读可能出现过度主观表达。应对：前置免责声明、保持"启发式"语言、增加边界提醒。

**产品风险**：功能过多导致主价值分散、内容太玄导致普通用户流失。应对：所有新功能优先服务主路径、每次版本只强化一个核心闭环。

**社交风险**：公开内容可能引发隐私与情绪安全问题。应对：先做轻分享再考虑轻互动、避免过早开放广场和评论。

---

## 13. 技术约束

纯前端单文件 SPA，无框架依赖，无构建工具，无后端服务（AI 解读除外）。数据持久化使用 IndexedDB（占卦记录）和 localStorage（用户设置）。部署于 Cloudflare Pages / GitHub Pages，支持纯静态托管。
