# 页面规格 V1

> 最后更新：2026-06-30
> 适用阶段：MVP 主路径收口 / UI 重构第一轮

## 1. 文档目的

本文件用于统一 `观心` 当前版本的页面层级、页面职责、主次路径与交互边界。

后续改动 `index.html` 时，优先遵循本文件，而不是按单个页面局部优化。

## 2. 产品主链路

当前产品主链路固定为：

`八卦为体，梅花为用，命宫 / 生日本命卦为人，五行为翻译层，节气为留存层`

对应到页面结构时，必须遵守以下原则：

1. 首页不承载所有解释，只承载“今天先从哪里开始”
2. `看自己` 负责认识个人底色，不负责重复首页和问卦结果
3. `问一事` 负责处理一个当下问题，不负责承担完整知识图鉴
4. `每日一卦` 负责日更陪伴和节奏，不和命宫页重复
5. 图鉴、历史、工具入口下沉，减少首页首屏噪音

## 3. 页面层级

### L0：启动与建档层

页面：
- 启动页
- 生辰录入页
- 建档结果过渡页

要求：
- 只给用户一个主动作
- 不在这一层灌输大量术语
- 录入成功后送入真正主路径

### L1：主导航层

L1 只保留三条主路径：

1. `每日一卦`
2. `看自己`
3. `问一事`

规则：
- 首页首屏只突出一个当天主入口
- 另外两条以次级入口存在
- 不在首页首屏并列展示过多解释性大卡片

## 4. 核心页面职责

### 首页

页面职责：
- 告诉用户今天先看什么
- 提供一个主入口和两个次入口
- 提供回看理由，而不是知识灌输

首页首屏只允许出现：
- 今日主线
- 命宫照面或今日练习中的一个核心摘要
- 一个主按钮
- 两个次按钮

首页不应该出现：
- 长篇术语解释
- 过多并列功能卡
- 图鉴 / 工具 / 历史的大面积入口

### 每日一卦页

页面职责：
- 承接“今日落点”
- 提供轻量日更陪伴

信息顺序：
1. 白话一句
2. 今日落点
3. 今日可做
4. 想学再展开原文

### 看自己 / 命宫档案页

页面职责：
- 解释“你平时怎么反应”
- 把命宫、本命卦、流年整理成个人档案

结构固定为四层：
1. `底色层`
2. `现实层`
3. `八星磁场层`
4. `术语与原文层`

要求：
- 首屏先给“这是什么人 / 这是什么惯性”
- 深读内容必须后置
- 四层切换必须明显可见

### 问一事页

页面职责：
- 围绕一个具体问题起卦并给出行动方向

结构固定为：
1. 起卦入口
2. 结果首屏
3. 深读折叠层
4. AI 追问层

结果首屏只保留：
- 先看卦在说什么
- 今日落点
- 先做什么
- 先别急什么

AI 追问规则：
- 默认不展开
- 只有用户先触发一次 AI 解读后才出现追问区
- 追问只围绕当前卦，不扩成泛心理咨询

## 5. 下沉页定位

### 八星磁场页

定位：
- 副观察层
- 用于性格磁场和数字组合学习

不再定位为：
- 主术数底层
- 首页主入口

### 卦象图鉴

定位：
- 学习页
- 从结果页精准跳入当前卦详情

### 历史 / 收藏 / 回看

定位：
- 回访页
- 不是单纯记录列表

必须强化：
- 日期分组
- 回看状态
- 主题聚合

## 6. 通用交互规则

### 返回

- 关键页面左上角固定返回
- 优先返回上一页
- 结果页返回不能再掉回摇签动画页
- 移动端允许右滑返回，但不能替代可见返回按钮

### 文本层级

所有解读遵守三层表达法：

1. `术数骨架`
2. `现实翻译`
3. `情绪承接`

禁止：
- 只有术语，没有白话
- 只有空泛安慰，没有卦意
- 所有卦都像同一个模板改写

### 模块显隐

- 默认先给结论
- 学习层后置
- AI 后置
- 工具后置

### 手机优先

- 首屏只保留最重要内容
- 长文本必须折叠或弹层化
- 模态层必须有稳定关闭入口

## 7. 当前 P0 收口范围

本轮恢复后，优先只做以下事项：

1. 首页首屏进一步收口成明确主次路径
2. 命宫档案页四层结构继续减重
3. 问一事结果页去重，强化结论与动作
4. AI 追问区维持“先触发再出现”
5. 图鉴、历史、工具入口继续下沉

本轮不做：

1. 新术数扩展
2. 微社交
3. 月度回顾扩容
4. 大规模拆文件

## 8. 页面验收口径

页面修改完成必须回答以下问题：

1. 用户一眼能不能分清这是什么页
2. 用户知不知道这一页先看什么
3. 用户知不知道看完下一步做什么
4. 用户能不能稳定返回
5. 页面是否仍然在服务“主路径”，而不是继续横向发散
