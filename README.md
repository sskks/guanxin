# 观心

一款以《周易》文化为灵感的单页 Web 应用，用更现代、轻量的交互方式，帮助用户通过卦象阅读、日常自省与 AI 白话解读来获得内在启发。

在线体验：[https://guanxin-1uu.pages.dev](https://guanxin-1uu.pages.dev)

## 产品定位

`观心` 不是传统意义上的工具型算命网站，而是一个以情绪陪伴为核心、以优秀传统文化为内容底座的东方文化产品。它把卦象、命宫、每日学习和 AI 解读组合在一起，降低普通用户接触《周易》的门槛，也帮助用户把学到的道理真正带回生活。

适合的人群：

- 对《周易》、卦象、传统文化感兴趣的用户
- 希望通过“每日一卦”进行自我观察和轻学习的用户
- 面对具体问题，想获得启发式建议和情绪安定感的用户

它更适合被理解为：

- `情绪类产品`
- `传统文化启发产品`
- `帮助用户调整状态、塑造更稳定三观的自我成长工具`

## 核心功能

- `每日一卦`
  按六十四卦顺序提供每日学习内容，支持卦象图解、卦辞、象辞与启示阅读，并带有打卡机制。

- `以事问卦`
  用户围绕具体问题起卦，通过竹筒摇签式交互获得本卦、互卦、变卦及结果解读。

- `AI 白话解读`
  基于 DashScope / Qwen 接口，将传统卦辞转换为更易理解的现代语言建议。

- `命宫档案`
  用户输入出生信息后，生成生日本命卦、命宫与流年参考内容，并以更适合回看的个人档案方式呈现；当前版本不承诺完整四柱八字排盘。

- `卦象图鉴`
  提供六十四卦检索、分类浏览和详情查看，适合做持续学习与查询。

- `数字起卦`
  支持用户输入数字快速起卦，满足不同使用偏好。

- `占卦历史`
  本地保存占卦记录，支持查看、搜索、收藏与回顾。

- `好友互占`
  支持生成分享链接，让好友查看指定卦象结果。

## 产品体验亮点

- 单文件 SPA，打开即用，部署成本低
- 整体视觉采用水墨、纸感、朱砂与太极意象，风格统一
- 交互围绕“观心”主题设计，强调仪式感与沉浸感
- 将传统文本、结构化卦象和 AI 解释结合，兼顾文化感与可理解性

## 技术方案

- `HTML + CSS + JavaScript` 单文件实现
- `IndexedDB` 保存占卦记录
- `localStorage` 保存用户设置、出生信息、打卡状态等本地数据
- `DashScope Compatible API` 调用 `qwen-plus` 生成 AI 解读
- 可部署到 `Cloudflare Pages` 等静态托管平台

## 架构演进方向

当前真实运行入口仍是根目录 `index.html`。为了支持后续 UI 升级、前后端能力和商业化，项目已建立并行架构骨架：

- `apps/web`：未来前端应用
- `apps/api`：未来后端服务
- `packages/design-system`：设计 token 与通用组件样式
- `packages/domain`：起卦、命宫、八星磁场等纯逻辑
- `packages/content`：白话解释、术语、AI prompt 与运营内容

这些目录目前是迁移目标，不会替代现有入口。后续开发会保持 `index.html` 可运行，并逐步抽离设计、逻辑和内容。

## 本地运行

当前版本推荐通过自带的 Python 服务启动，这样静态页面和 AI 代理接口可以一起工作。

```powershell
cd D:\vscode\Microsoft VS Code\project\Cculture
$env:DASHSCOPE_API_KEY="你的 DashScope Key"
python server.py
```

然后访问：[http://127.0.0.1:8080](http://127.0.0.1:8080)

## AI 解读配置

AI 解读现在通过本地后端代理访问 DashScope。

- 服务端优先从环境变量 `DASHSCOPE_API_KEY` 读取 Key
- 如果你不方便配环境变量，也可以在页面右上角“设置”里临时填写当前浏览器使用的 Qwen Key
- 前端只向同源接口 `/api/ai-interpret` 发起请求
- 本地填写的 Key 仅保存在当前浏览器，并通过本地代理转发，不会直接暴露到第三方页面

如果你计划公开部署，建议继续保持这种模式：

- 由后端代理调用大模型接口
- 通过部署环境变量注入 Key

## 数据与隐私

当前版本主要使用浏览器本地存储能力：

- 出生信息存储在本地 `localStorage`
- 占卦记录存储在本地 `IndexedDB`
- 打卡、收藏等信息也保存在本地浏览器

这意味着：

- 用户更换浏览器或清理本地数据后，记录可能丢失
- 当前版本不提供跨设备同步
- 涉及出生信息与个人问题内容时，仍应在产品中明确展示隐私说明

## 免责声明

本项目用于传统文化学习、内容体验与启发式参考，不构成医疗、法律、投资、心理治疗等专业建议。用户应将卦象与 AI 解读视为辅助理解与自我观察工具，而非绝对决策依据。

## 当前版本更像什么

从产品形态上看，`观心` 当前更接近：

- `传统文化内容产品`
- `轻量情绪陪伴产品`
- `带有仪式感的自我探索工具`

而不是高频工具型应用。这意味着它更依赖：

- 首次体验是否顺滑
- 内容是否可信且易懂
- 用户是否愿意形成“每日一卦”的使用习惯

## 后续优化方向

- 明确首页主入口优先级，减少首次使用决策成本
- 强化新手教育，降低“命宫 / 起卦 / 图鉴”之间的理解门槛
- 优化 AI 解读可信感，增加免责声明与结果边界说明
- 增加连续打卡、收藏主题、月度回顾等留存机制
- 如需长期运营，可补充账号体系与多端同步
